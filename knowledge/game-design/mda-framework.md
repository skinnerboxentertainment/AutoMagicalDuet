# MDA Framework

Design mechanics from the desired player experience backward. State the target aesthetics first, then identify the dynamics those feelings require, then specify mechanics that reliably create those dynamics.

## Patterns

- Start with the emotional target: challenge, discovery, expression, fantasy, fellowship, narrative, sensation, or submission.
- Describe expected dynamics as player behaviors and emergent patterns, not implementation details.
- Define mechanics as precise rules that generate those dynamics.
- Validate mechanics against Self-Determination Theory: autonomy, competence, and relatedness.
- Use playtest acceptance criteria to verify both function and feel.

## Anti-patterns

- Beginning with a system because it is familiar rather than because it serves the target aesthetic.
- Treating "fun" or "feels good" as sufficient specification.
- Adding mechanics that create dynamics opposed to the project pillars.
- Shipping choices where one option clearly dominates while pretending they support autonomy.
